# 微信 / 腾讯内容搜索

多引擎搜索腾讯体系内容（微信公众号、腾讯新闻等 qq.com 域名），Playwright 浏览器获取真实链接和内容，支持深度调研。

## 用法

```
/weixin-search <关键词>
/weixin-search --pages 3 AI大模型
/weixin-search --time week 新能源汽车
/weixin-search --deep Claude Code 最新功能
```

## 参数解析

用户输入：`{{ input }}`

| 参数 | 默认值 | 说明 | 示例 |
|------|--------|------|------|
| `<关键词>` | （必填） | 搜索关键词 | `AI大模型` |
| `--pages N` | 普通 `3` / 深度 `5` | 每个引擎的搜索页数 | `--pages 5` |
| `--time RANGE` | 无限制 | 时间范围：`day`/`week`/`month`/`year` | `--time week` |
| `--deep` | 关闭 | 启用深度调研模式 | `--deep` |

解析规则：
- **无参数**：提示用户输入搜索关键词
- **仅关键词**：`/weixin-search AI大模型` → 快速搜索（搜狗+百度）
- **指定页数**：`/weixin-search --pages 5 AI大模型` → 每个引擎搜 5 页
- **指定时间**：`/weixin-search --time week 新能源` → 只看最近一周
- **深度调研**：`/weixin-search --deep AI大模型` → 全引擎 + Tavily 调研
- 参数可组合：`/weixin-search --pages 2 --time month AI大模型`

## 搜索引擎说明

| 引擎 | 特点 | 何时使用 |
|------|------|---------|
| **搜狗微信** | 微信独家索引，覆盖最全，结果最精准 | 始终使用（主力） |
| **百度** | 最大中文索引，`site:qq.com` 覆盖腾讯全域 | 始终使用（补充） |
| **Google** | 国际视角，能找到长尾/冷门文章 | 深度调研模式使用 |
| **Tavily Search** | AI 搜索引擎，直接返回 qq.com 域名链接 | 深度调研 + 兜底 |
| **Tavily Research** | 多源综合调研，不限微信 | 深度调研模式使用 |

## 智能搜索策略

搜索不是"执行一次命令就结束"，而是一个**推理-搜索-评估-调整的循环**。以下策略贯穿整个搜索过程。

### 1. 查询理解与构造

在执行搜索前，先理解用户的真实意图，构造最优搜索关键词：

- **提取核心概念**：从用户输入中识别关键实体、事件、时间范围
  - 用户说"最近充电桩行业有什么政策变化" → 关键词 `充电桩 政策 2025`
- **补充隐含上下文**：利用对话历史补全用户省略的信息
  - 如果上文在讨论新能源汽车，用户说"搜一下补贴" → 关键词 `新能源汽车 补贴政策`
- **调整术语精度**：根据用户的专业程度选择搜索用语
  - 技术用户 → 用行业术语（如"V2G 车网互动"）
  - 普通用户 → 用通俗表述（如"电动车给电网充电"）

### 2. 查询重构（Query Reformulation）

**当搜索结果不理想时（结果少于 3 条、相关度低、覆盖面窄），自动重构查询并重试：**

| 策略 | 触发条件 | 示例 |
|------|---------|------|
| **拆解子问题** | 复合查询结果差 | "固态电池量产时间线和技术路线" → 先搜"固态电池量产"，再搜"固态电池技术路线" |
| **换同义词** | 首次搜索结果少 | "新能源车" ↔ "电动汽车" ↔ "NEV" |
| **加限定词** | 结果太泛 | "AI" → "AI大模型 2025" / "AI 医疗应用" |
| **去限定词** | 结果太少 | "2025年Q2充电桩政策" → "充电桩政策" |
| **切换语言** | 技术/学术主题 | 中文"大语言模型" → 英文"LLM large language model" |
| **换视角** | 单一来源 | "充电桩技术" → "充电桩 测评" / "充电桩 用户体验" |
| **追踪线索** | 某篇文章提到关键信息 | 文章提到"独角兽计划" → 追加搜索"小桔充电 独角兽计划" |

重构后的搜索**累积合并**之前的结果，不丢弃已有数据。

### 3. 结果质量评估

每次搜索后评估结果质量，决定是否需要进一步搜索：

- **数量充足？** — 至少 5 条相关结果才算合格
- **相关度高？** — 标题和用户意图匹配，不是打擦边球的结果
- **来源多样？** — 不全是同一个公众号或同一个主题的重复内容
- **时效性满足？** — 用户关心最新动态时，结果不能全是几年前的
- **覆盖面够？** — 不同角度/立场都有涉及（正面/负面/中性）

如果评估不通过，回到"查询重构"步骤，**最多迭代 3 轮**。

### 4. 内容智能分析

获取文章内容后，不是简单复述原文，而是：

- **结构化提取**：识别文章中的关键数据、核心观点、时间节点、人物引用
- **关联用户意图**：重点摘取与用户搜索目的相关的段落，省略无关内容
- **标注信息质量**：区分事实（"2025年Q1充电桩保有量达1200万台"）和观点（"预计将突破2000万台"）
- **交叉验证**：多篇文章中相同数据相互印证的，标注为"多源确认"；仅一家之言的，标注为"单一来源"
- **识别信息缺口**：指出搜索结果未覆盖的方面（"未找到关于成本数据的信息"）

### 5. 主动建议与引导

搜索完成后，基于结果主动为用户提供下一步建议：

- **发现延伸线索**："这篇文章提到了 XX 政策，要不要深入了解？"
- **指出信息缺口**："目前找到的都是行业分析，缺少技术细节，需要我换个角度搜一下吗？"
- **推荐深入阅读**："第 3 篇和第 7 篇质量最高，建议优先阅读"
- **建议深度模式**："快速搜索只覆盖了部分信息，这个主题适合用 `--deep` 做深度调研"

## 执行流程

### 模式一：快速搜索（默认）

#### Step 1: 多引擎搜索

```bash
weixin-search search "<关键词>" --pages <N> --engines sogou,baidu
```

- `--pages` 取用户指定值，未指定则默认 `3`
- 使用 Playwright 无头浏览器同时搜索搜狗微信和百度
- 输出 JSON 数组，每条包含 `title`、`link`、`real_url`、`publish_time`、`source`
- 结果自动按标题去重

#### Step 1.5: 时间过滤

如果用户指定了 `--time`，对搜索结果按 `publish_time` 过滤：
- `day`：只保留最近 24 小时
- `week`：只保留最近 7 天
- `month`：只保留最近 30 天
- `year`：只保留最近 365 天

#### Step 2: 结果质量评估与迭代

评估搜索结果（见"结果质量评估"章节）。如果不满足要求：

1. **结果为零**：使用 Tavily 兜底（见 Fallback 章节）
2. **结果少或相关度低**：按"查询重构"策略换关键词重试，重试结果与已有结果合并
3. **结果充足**：跳到 Step 3

**重要**：迭代时向用户简要说明正在做什么（如"结果较少，正在用 XX 关键词补充搜索…"），不要静默重试。

#### Step 3: 展示结果

以表格形式展示搜索结果：

| # | 标题 | 发布时间 | 来源 |
|---|------|---------|------|
| 1 | 文章标题 | 2025-05-04 | sogou |
| 2 | 文章标题 | 2025-05-03 | baidu |

展示后：
- 简要总结搜索结果的整体情况（覆盖了哪些方面、可能缺失什么）
- 标注推荐阅读（如果能从标题判断质量）
- 询问用户：「找到 N 篇文章，需要查看哪篇的详细内容？输入序号即可。」

#### Step 4: 获取文章内容

当用户选择某篇文章后，使用 Playwright 获取正文：

```bash
weixin-search content "<url>"
```

- `<url>` 优先使用 `real_url`，如为空则使用 `link`
- 脚本自动处理各种 URL 类型：
  - `qq.com` 域名链接：直接获取（微信公众号、腾讯新闻等）
  - 搜狗/百度重定向链接：自动解析跳转后获取
  - 文章已删除：自动尝试 web.archive.org 存档
- 如果 Playwright 获取失败，fallback 到 Tavily Extract：

使用 `mcp__tavily__tavily_extract` 工具：
- `urls`: `["<real_url>"]`
- `extract_depth`: `"advanced"`
- `query`: 用户原始搜索关键词
- `format`: `"markdown"`

#### Step 5: 内容分析与建议

获取内容后：

1. **结构化摘要**：提取核心观点、关键数据、时间节点（见"内容智能分析"章节）
2. **关联搜索意图**：重点分析与用户搜索关键词相关的部分
3. **主动建议**：根据文章内容推荐后续动作（见"主动建议与引导"章节）
   - 文章引用了其他来源 → 提供追踪搜索
   - 内容中有专业术语 → 解释术语
   - 文章提到相关事件/政策 → 建议延伸搜索

### 模式二：深度调研（--deep）

当用户使用 `--deep` 参数时，执行全面调研：

#### Step 1: 全引擎并行搜索

同时执行：

1. **Playwright 多引擎搜索**（搜狗 + 百度 + Google）：
```bash
weixin-search search "<关键词>" --pages <N> --engines sogou,baidu,google
```
- `--pages` 默认 `5`

2. **Tavily Search**（微信域名限定）：
使用 `mcp__tavily__tavily_search` 工具：
- `query`: 用户关键词
- `search_depth`: `"advanced"`
- `include_domains`: `["qq.com"]`
- `max_results`: 20
- `time_range`: 取用户 `--time` 值，未指定则默认 `"month"`

3. **Tavily Research**（综合调研，不限微信）：
使用 `mcp__tavily__tavily_research` 工具：
- `input`: `"调研以下主题，重点关注微信公众号和腾讯新闻的观点和数据：<关键词>"`
- `model`: `"pro"`

合并所有搜索结果，按标题去重。

**如果合并后结果仍不充分，应用"查询重构"策略**：拆解子问题、换同义词、切换语言，用 Tavily Search 或 Playwright 补充搜索，直到信息覆盖面满足调研需求。

#### Step 2: 深入阅读关键文章

从结果中选择最相关的 3-5 篇文章，获取全文：

- 有 `real_url` 的文章：使用 Playwright 本地获取
- 仅有 Tavily URL 的文章：使用 `mcp__tavily__tavily_extract` 获取
  - `urls`: 选取的文章 URL 列表
  - `extract_depth`: `"advanced"`
  - `query`: 用户关键词

选择文章的优先级：
- 时效性高的优先（最近发布的）
- 数据丰富的优先（含统计、图表、案例）
- 来源权威的优先（官方公众号、行业知名媒体）
- 覆盖不同角度（避免全部选同一立场的文章）

#### Step 3: 发现更多内容（可选）

如果某篇文章质量很高，使用 `mcp__tavily__tavily_crawl` 从该文章出发发现同一公众号的更多文章：
- `url`: 高质量文章的 URL
- `instructions`: `"查找同一公众号下与<关键词>相关的其他文章"`
- `max_depth`: 2
- `limit`: 10
- `select_domains`: `["qq.com"]`

如果阅读过程中发现新线索（文章提到的事件、引用的数据来源、提及的关键人物/企业），**追加搜索这些线索**。

#### Step 4: 输出调研报告

整合所有信息，输出结构化报告：

- **主题概述**：一段话概括调研发现
- **核心观点**：按文章来源标注，区分事实和观点
- **关键数据/事实**：多源交叉验证的标注为"多源确认"
- **不同声音**：如果有分歧或争议，客观呈现各方立场
- **趋势/结论**：基于多篇文章的综合判断
- **信息缺口**：指出调研未覆盖的方面，建议后续研究方向
- **参考文章列表**：标题 + 链接 + 一句话评价

## Fallback: 多层错误恢复

搜索和内容获取的每个环节都有容错策略：

### 搜索兜底

当 Playwright 搜索返回空结果时：

**第一层：Tavily 搜索**

使用 `mcp__tavily__tavily_search` 工具：
- `query`: 用户关键词
- `include_domains`: `["qq.com"]`
- `search_depth`: `"advanced"`
- `max_results`: `--pages` × 10（默认 30）
- `time_range`: 取用户 `--time` 值（如有）

**第二层：查询重构后重试**

如果 Tavily 也没有结果，自动应用"查询重构"策略换关键词重试。

**第三层：扩大搜索范围**

去掉 `include_domains` 限制，在全网搜索后筛选 qq.com 结果。

### 内容获取兜底

当无法获取文章正文时：

1. **Playwright 直接获取** → 失败
2. **Tavily Extract**（`extract_depth: "advanced"`）→ 失败
3. **web.archive.org 存档** → 失败
4. **告知用户并提供替代方案**：搜索同主题的其他可达文章，而非简单报错

## 注意事项

- 搜索使用 Playwright 无头浏览器，搜狗+百度双引擎默认开启，深度模式加入 Google
- 多引擎并行搜索（每个引擎独立 tab），搜索耗时约 15-30 秒
- 百度结果通过 `mu` 属性直接获取真实 qq.com 链接，无需额外跳转
- 搜狗/百度重定向链接在搜索阶段就已解析为真实 URL，内容获取无障碍
- 已删除的文章会自动尝试 web.archive.org 存档获取
- 深度调研模式叠加 Tavily 多源搜索 + Research，覆盖面最广但耗时更长
- 迭代搜索时始终告知用户当前状态，避免长时间静默
